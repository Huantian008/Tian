# C++ 代码目录说明

这个目录用来集中存放我的 C++ 学习代码，主要包括教材示例、基础练习、OJ 题目和 LeetCode 刷题记录。整体目标很简单：源码按类型分类，编译产物单独放，平时打开目录就能快速找到题目和可执行文件。

## 当前结构

```text
cpp/
├── .vscode/         VS Code 的构建、运行、调试配置
├── book-of-cpp/     教材或教程里的示例代码
├── exercises/       平时练习题
├── LeetCode/        LeetCode 题解
├── oj/              OJ 平台练习题
├── output/          根目录下源码的编译输出目录
└── README.md
```

说明：

- `book-of-cpp/` 里的文件以章节编号为主，适合回头查基础语法和示例。
- `exercises/`、`LeetCode/`、`oj/` 更偏向题目练习。
- 各子目录里的 `output/` 通常是编译后生成的 `.exe`，不属于核心源码。

## 目录使用规则

- 源码统一保留在对应分类目录下，文件后缀以 `.cpp` 为主。
- 编译输出默认放到当前源码目录下的 `output/` 文件夹。
- `output/` 可以随时清空；删掉后重新编译会自动生成。
- 这个目录目前基本不放通用头文件库，主要是单文件练习代码。

## 编译与运行

当前工作区已经配置好 VS Code 任务，编译器使用：

```text
E:\MinGW\ucrt64\bin\g++.exe
```

编译参数为：

```text
-g -std=c++17
```

### 在 VS Code 中使用

可直接使用下面两个调试配置：

- `C++: Debug active file (Windows)`
- `C++: Run active file (Windows)`

或者使用默认构建任务：

- `C/C++: g++.exe 生成活动文件`

执行后会在当前源文件所在目录下自动创建 `output/`，并生成同名 `.exe`。

### 手动编译

如果不走 VS Code，也可以在对应目录里直接执行：

```powershell
E:\MinGW\ucrt64\bin\g++.exe -g -std=c++17 .\xxx.cpp -o .\output\xxx.exe
```

第一次编译前如果没有 `output/`，先手动创建即可。

## 内容分类建议

- 教材例题放 `book-of-cpp/`
- 自己写的小练习放 `exercises/`
- 在线判题平台题目放 `oj/`
- LeetCode 题目放 `LeetCode/`

后续如果题量继续增加，再考虑按专题或题号范围继续拆分子目录。

## 维护原则

- 保留源码，少保留中间产物。
- 目录名尽量按用途划分，不按临时想法随意堆文件。
- 新写的题目优先放到已经存在的分类中，避免根目录变乱。
- 如果只是重新编译，优先清 `output/`，不要误删源码文件。
